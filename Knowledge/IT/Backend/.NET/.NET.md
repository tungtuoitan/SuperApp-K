# .NET là gì? [id:2690 order:1]
là nền tảng phát triển phần mềm của Microsoft, gồm runtime (CLR) và thư viện chuẩn (BCL), dùng để build app console, web, desktop, mobile.

# namespace trong .NET là gì? [id:2797 order:2]
là 1 nhóm các class/interface

# assembly là gì? [id:2798 order:3]
là đơn vị deploy của .NET — thường là file `.dll` hoặc `.exe`, chứa IL và metadata của 1 project hoặc library.
# sự khác nhau giữa dll và exe?
`exe` là entry point — chạy được trực tiếp, có hàm `Main`. 
`dll` là library — không chạy độc lập, được app khác load và gọi.
# dll nghĩa là gì?
là Dynamic Link Library — thư viện liên kết động, được load vào memory lúc app chạy thay vì gộp vào file `exe` lúc build.
# khi nào cần thì exe mới load dll, chứ k phải load hết lúc ban đầu à ?
Đúng. CLR load `dll` lazy — khi code lần đầu reference type bên trong dll đó, CLR mới đọc và map vào memory. Tiết kiệm RAM và startup time.
# file web.config là gì?
là file config cũ, gần như không dùng, thay bằng `appsettings.json`.
# sau khi build, folder cs,de,es,fr.... là gì?
là satellite assemblies cho từng ngôn ngữ — chứa resource đã localize (`cs` = Czech, `de` = German, `es` = Spanish, `fr` = French...). .NET load đúng folder theo culture.
# file x.csproj là gì?
là file project của C# 
# file project dùng làm gì?
— khai báo SDK, target framework, package reference, build setting. 
# file project được dùng thế nào?
Dùng bởi MSBuild/`dotnet build`.
# so sánh Program.cs và Startup.cs?
- `Program.cs`: entry point, có `Main` (hoặc top-level statements). Tạo và chạy host.
- `Startup.cs`: nơi config services và middleware (`ConfigureServices`, `Configure`). Từ .NET 6+ thường gộp hết vào `Program.cs`.
# .NET hiện đại thường k dùng startup nữa mà viết hết vào program à?
Đúng. Từ .NET 6+ template mặc định bỏ `Startup.cs`, gộp services + middleware vào `Program.cs` dùng top-level statements. Project cũ vẫn giữ `Startup.cs` được.
# lanchSettings khác gì appsettings
- `launchSettings.json`: chỉ dùng khi dev local — config IDE chạy app (URL, env vars, profile). Không deploy.
- `appsettings.json`: config app runtime (DB connection, logging) — đi kèm khi deploy.
# xAPI project trong 1 solution gọi là gì?
là Web/API project 
— project loại `Microsoft.NET.Sdk.Web`, host endpoint HTTP. Thường có `Program.cs`, controller, `appsettings.json`.
# endpoint có những nghĩa nào?
- API endpoint: 1 URL + HTTP method handler (`GET /users/1`)
- Network endpoint: cặp IP:port của 1 connection
- Service endpoint: địa chỉ public của 1 service (Azure, AWS)
- ASP.NET Endpoint: object đại diện route đã match, do routing middleware tạo
# sau khi build thì mọi project đều có bin và obj folder à?
Đúng
# bin và obj là gì?
- `bin`: chứa output cuối — file `.dll`/`.exe` để chạy hoặc deploy.
- `obj`: chứa file trung gian (intermediate) trong quá trình build — IL chưa link, cache, metadata. Build xong có thể xoá.
# file exe sẽ link tới tất cả file dll trong bin ở tất cả project à?
Không link sẵn tất cả. `exe` chỉ reference các `dll` mà code thực sự dùng (khai báo trong `csproj`). Khi build, MSBuild copy dll cần thiết vào `bin` của exe. Lúc chạy, CLR load lazy theo nhu cầu.
# MSBuild là gì?
là build engine của Microsoft — chạy file project (`.csproj`) để compile, link, package. `dotnet build` gọi MSBuild bên dưới.
# vai trò của MSBuild?
- Đọc `.csproj` và xác định task cần làm
- Compile source thành IL
- Resolve dependency và copy dll vào `bin`
- Chạy custom target (publish, pack, ...)
# đơn vị deploy là gì? [id:2799 order:4]
là phần nhỏ nhất mà bạn copy/ship lên server để app chạy. Trong .NET là file `.dll`/`.exe`.

# đơn vị deploy hầu hết là file phải không? [id:2800 order:5]
Đúng, nhưng cũng có thể là folder, container image (Docker), hoặc package (`.nupkg`, `.zip`) tùy cách ship.

# ship là gì? [id:2801 order:6]
là đóng gói và đưa code/sản phẩm lên môi trường thật để user dùng — có thể là deploy server, publish package, push image lên registry.

# value type và reference type khác nhau gì?
Value type lưu giá trị ở stack hoặc inline trong object. 
Reference type lưu địa chỉ trỏ đến object trên heap
