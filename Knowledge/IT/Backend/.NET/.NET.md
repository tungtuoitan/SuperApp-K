# .NET là gì?
là nền tảng phát triển phần mềm của Microsoft, gồm runtime (CLR) và thư viện chuẩn (BCL), dùng để build app console, web, desktop, mobile.

# CLR là gì?
là runtime của .NET
, chịu trách nhiệm biên dịch IL sang machine code (JIT), quản lý bộ nhớ (GC), và xử lý exception.
# runtime gồm những gì?
gồm: JIT compiler, GC, type system, thread scheduler, exception handler, và BCL (thư viện chuẩn).
# IL là gì?
là Intermediate Language — bytecode mà compiler C# tạo ra. CLR sẽ JIT compile IL thành machine code lúc runtime.
# compiler tạo ra gì?
tạo ra IL (file `.dll` hoặc `.exe`) cùng metadata, không phải machine code trực tiếp.
# tại sao tại cần compiler?
# CLR tạo ra gì?
tạo ra machine code
# JIT là gì?
là quá trình CLR biên dịch IL sang machine code

# GC trong .NET là gì?
là Garbage Collector — tự động giải phóng memory của object không còn được tham chiếu, không cần dev tự `free()`.

# interface khác abstract class thế nào trong C#?
- Interface: chỉ định nghĩa contract (method/property), không có implementation (từ C# 8 có default impl)
- Abstract class: có thể có implementation, chỉ kế thừa được 1 class, thêm trạng thái (field)
# contract có nghĩa là gì?
là cam kết về những method/property mà class phải có, không quy định cách implement.

# IDisposable và using statement dùng để làm gì?
`IDisposable` là interface để giải phóng tài nguyên unmanaged (file, connection, socket). `using` block tự gọi `Dispose()` khi ra khỏi scope.
# tài nguyên unmanaged nghĩa là gì?
là tài nguyên do OS quản lý chứ không phải CLR — file handle, network socket, DB connection. GC không tự dọn được, phải `Dispose()`.

# Task và Thread khác nhau thế nào?
Thread là đơn vị OS thực sự, tốn tài nguyên. Task là abstraction cao hơn, nhiều Task chạy trên ít thread qua thread pool — nhẹ hơn và dễ compose hơn.
# abstraction cao hơn nghĩa là gì? cho ví dụ phổ biến
# đơn vị OS là gì?
là tài nguyên thực do OS cấp phát và quản lý — process, thread, file handle, socket.
# abstraction nghĩa là gì?
là che giấu chi tiết phức tạp bên dưới, chỉ phơi ra interface đơn giản. Task abstract Thread, ORM abstract SQL.

# compose là gì?
là ghép nhiều thành phần nhỏ lại thành thành phần lớn hơn. 
Trong .NET, compose Task là chain `await` nhiều operation, hoặc dùng `Task.WhenAll`.

# dependency injection là gì?
là 1 pattern
# DI giải quyết vấn đề gì?
- Tight coupling: class tự `new` dependency
- Khó test: không inject được mock
- Khó thay đổi: muốn đổi implementation phải sửa nhiều chỗ
# DI có phải là technique không?
Có. DI vừa là pattern (cách tổ chức) vừa là technique (cách code).
# singleton, scoped, transient là gì?
là 3 lifetime của service trong DI container — quy định khi nào instance được tạo mới.
# Singleton, Scoped, Transient khác nhau thế nào?
- Singleton: 1 instance dùng suốt lifetime app
- Scoped: 1 instance mỗi HTTP request
- Transient: tạo mới mỗi lần inject

# kestrel có phải là http server không?

# mỗi middleware là 1 class độc lập à?
